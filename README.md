# Novella — Система поиска и заказа книг в библиотеке

## Описание проекта

Novella — это современная веб-система для поиска, просмотра и онлайн-бронирования книг в университетской/городской библиотеке. Проект разработан как курсовая работа по дисциплине «Проектирование веб-сайта и обеспечение его бесперебойной работы».

### Основные возможности

- 🔍 **Поиск книг** по названию, автору, жанру, году издания
- 📖 **Просмотр детальной информации** о книгах с обложками и описаниями
- 📅 **Онлайн-бронирование** книг без посещения библиотеки
- 👤 **Личный кабинет** пользователя с историей бронирований
- 🔐 **Регистрация и авторизация** пользователей
- ⭐ **Отзывы и рейтинги** к книгам
- 📱 **Адаптивный дизайн** для всех устройств
- 🛡️ **Админ-панель** для управления книгами

## Технологический стек

### Frontend
- **React 18** — функциональные компоненты, хуки
- **React Router** — навигация между страницами
- **Tailwind CSS** — стилизация
- **Axios** — HTTP-клиент для API

### Backend
- **Node.js + Express** — REST API
- **JWT** — аутентификация
- **bcrypt** — хэширование паролей
- **CORS** — кросс-доменная поддержка

### База данных
- **MongoDB** (рекомендуется для продакшена)
- **In-memory storage** (для разработки)

### Развертывание
- **Vercel** — frontend
- **Render/Railway** — backend
- **Git** — система контроля версий

## Установка и запуск

### Предварительные требования

- Node.js 16+
- npm или yarn
- Git

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/novella.git
cd novella
```

### 2. Установка зависимостей

#### Backend
```bash
cd backend
npm install
```

#### Frontend
```bash
cd ../frontend
npm install
```

### 3. Настройка переменных окружения

Создайте файл `.env` в папке `backend`:

```env
PORT=5000
JWT_SECRET=your-super-secret-jwt-key-here
TOKEN_EXPIRY=1h
```

### 4. Запуск в режиме разработки

#### Backend (в первом терминале)
```bash
cd backend
npm run dev
```

#### Frontend (во втором терминале)
```bash
cd frontend
npm run dev
```

Приложение будет доступно по адресу:
- Frontend: http://localhost:5173
- Backend API: http://localhost:5000

## API документация

### Аутентификация

#### POST /api/auth/register
Регистрация нового пользователя.

**Запрос:**
```json
{
  "name": "Иван Иванов",
  "email": "ivan@example.com",
  "password": "password123"
}
```

**Ответ:**
```json
{
  "message": "Регистрация прошла успешно",
  "token": "jwt-token-here",
  "user": {
    "id": "1",
    "name": "Иван Иванов",
    "email": "ivan@example.com",
    "role": "user"
  }
}
```

#### POST /api/auth/login
Вход в систему.

**Запрос:**
```json
{
  "email": "ivan@example.com",
  "password": "password123"
}
```

### Книги

#### GET /api/books
Получение списка книг с фильтрами.

**Параметры запроса:**
- `q` — поиск по названию/автору
- `author` — фильтр по автору
- `genre` — фильтр по жанру
- `year` — фильтр по году
- `status` — фильтр по статусу (available/booked)

#### GET /api/books/:id
Получение детальной информации о книге.

#### POST /api/books/:id/book
Бронирование книги (требуется авторизация).

#### POST /api/books/:id/review
Добавление отзыва к книге (требуется авторизация).

**Запрос:**
```json
{
  "rating": 5,
  "comment": "Отличная книга!"
}
```

## Развертывание

### Frontend (Vercel)

1. Создайте аккаунт на [Vercel](https://vercel.com)
2. Подключите репозиторий GitHub
3. Установите переменную окружения `VITE_API_BASE` с URL backend API
4. Деплой автоматически запустится при пуше в main

### Backend (Render)

1. Создайте аккаунт на [Render](https://render.com)
2. Создайте новый Web Service
3. Подключите репозиторий GitHub
4. Установите переменные окружения (JWT_SECRET, etc.)
5. Установите команду сборки: `npm install`
6. Установите команду запуска: `npm start`

## Тестирование

### Ручное тестирование

1. **Регистрация и вход:**
   - Создайте новый аккаунт
   - Войдите в систему
   - Проверьте сохранение сессии

2. **Каталог книг:**
   - Просмотрите список книг
   - Используйте фильтры поиска
   - Проверьте пагинацию

3. **Бронирование:**
   - Выберите доступную книгу
   - Забронируйте её
   - Проверьте статус в личном кабинете

4. **Отзывы:**
   - Добавьте отзыв к книге
   - Проверьте отображение отзывов

### API тестирование

Используйте Postman или аналогичный инструмент для тестирования API:

1. **Регистрация:** POST /api/auth/register
2. **Вход:** POST /api/auth/login
3. **Получение книг:** GET /api/books
4. **Бронирование:** POST /api/books/1/book (с заголовком Authorization)
